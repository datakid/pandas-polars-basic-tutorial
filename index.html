Installation & Setup

Both libraries install via pip. Polars ships with zero hard dependencies; Pandas benefits significantly from PyArrow for its new string backend and zero-copy conversions.

Pandas 3.0

pip install "pandas>=3.0"
pip install pyarrow

PyArrow enables the native str dtype, zero-copy Arrow conversions, and GPU compute backend.

Polars 1.x

pip install "polars>=1.0"
pip install "polars[all]"

polars[all] adds S3/GCS/Azure I/O, timezone support, and the built-in SQL engine.

Verify Installation

Verify
import pandas as pd

03

Reading & Writing Data

Polars uses scan_* for lazy reads — nothing loads until .collect(). Pandas reads eagerly. Parquet is the gold standard: columnar, compressed, dtype-preserving.

CSV

Pandas
df = pd.read_csv("data.csv")
df = pd.read_csv(
    "data.csv",
    usecols=["id", "amount"],
    dtype={"id": "int32"},
    parse_dates=["created_at"],
    na_values=["N/A", "—"],
    chunksize=100_000,
)
df.to_csv("out.csv", index=False)
Polars
df = pl.read_csv("data.csv")
lf = pl.scan_csv("data.csv")
df = pl.read_csv(
    "data.csv",
    columns=["id", "amount"],
    schema_overrides={"id": pl.Int32},
    try_parse_dates=True,
    null_values=["N/A", "—"],
)
df.write_csv("out.csv")

Parquet

Pandas
df = pd.read_parquet("data.parquet")
df = pd.read_parquet(
    "data.parquet",
    columns=["id", "amount"],
    filters=[("region", "==", "EU")],
)
df.to_parquet(
    "out.parquet",
    compression="zstd",
    index=False,
)
Polars
lf = pl.scan_parquet("data.parquet")
lf = pl.scan_parquet(
    "s3://bucket/events/year=*/**.parquet",
    hive_partitioning=True,
)
df = pl.read_parquet("data.parquet")
df.write_parquet("out.parquet", compression="zstd")

JSON / NDJSON / Excel

Pandas
df = pd.read_json("data.json")
df = pd.read_json("data.ndjson", lines=True)
df = pd.read_excel(
    "data.xlsx",
    sheet_name="Sales",
    skiprows=1,
)
df.to_excel("out.xlsx", index=False)
Polars
df = pl.read_json("data.json")
df = pl.read_ndjson("data.ndjson")
lf = pl.scan_ndjson("data.ndjson")
df = pl.read_excel(
    "data.xlsx",
    sheet_name="Sales",
)
df.write_excel("out.xlsx")

Parquet over CSV: A 1 GB CSV typically becomes ~90 MB Parquet with zstd, reads 15–30× faster, and preserves dtypes exactly. Both libraries push filters and column selection into the file reader — only the bytes you need are ever loaded.

Selecting & Filtering

Polars uses a pure expression API — no .loc/.iloc confusion, no bracket ambiguity. Pandas 3.0 makes slicing fully safe with mandatory Copy-on-Write.

Select columns

Pandas
df["name"]
df[["name", "score"]]
df.loc[:, "name":"score"]
df.iloc[:, 0]
df.iloc[:, [0, 2]]
df.filter(regex="^score")
df.select_dtypes(include="number")
Polars
df["name"]
df.select(["name", "score"])
df[:, 0]
df.select(pl.first())
df.select(pl.last())
df.select(pl.col("^score.*$"))
df.select(pl.col(pl.Int64, pl.Float64))

Filter rows

Pandas
df[df["age"] > 25]
df[(df["age"] > 25) &
   (df["score"] >= 90)]
df.query("age > 25 and score >= 90")
df[df["city"].isin(["Paris", "Lyon"])]
df[~df["city"].isin(["Paris", "Lyon"])]
df[df["name"].notna()]
Polars
df.filter(pl.col("age") > 25)
df.filter(
    (pl.col("age") > 25) &
    (pl.col("score") >= 90)
)
df.filter(
    pl.col("city").is_in(["Paris", "Lyon"])
)
df.filter(
    ~pl.col("city").is_in(["Paris", "Lyon"])
)
df.filter(pl.col("name").is_not_null())

Slicing & sampling

Pandas

05

Transforming Columns

Polars' with_columns evaluates all expressions in parallel. Pandas 3.0's assign chains cleanly with Copy-on-Write — no more accidental mutations to the original DataFrame.

Add / update columns

Pandas
df["bonus"] = df["salary"] * 0.10

df = df.assign(
    bonus  = lambda x: x["salary"] * 0.10,
    grade  = lambda x: x["score"] // 10,
    senior = lambda x: x["age"] >= 40,
)
Polars
df = df.with_columns(
    (pl.col("salary") * 0.10).alias("bonus"),
    (pl.col("score") // 10).alias("grade"),
    (pl.col("age") >= 40).alias("senior"),
)

Conditional logic

Pandas
import numpy as np

df["tier"] = np.where(
    df["revenue"] > 1000, "High", "Low"
)

conditions = [
    df["score"] >= 90,
    df["score"] >= 75,
    df["score"] >= 60,
]
choices = ["A", "B", "C"]
df["grade"] = np.select(conditions, choices, default="F")
Polars
df = df.with_columns(
    pl.when(pl.col("revenue") > 1000)
      .then(pl.lit("High"))
      .otherwise(pl.lit("Low"))
      .alias("tier")
)
df = df.with_columns(
    pl.when(pl.col("score") >= 90).then(pl.lit("A"))
      .when(pl.col("score") >= 75).then(pl.lit("B"))
      .when(pl.col("score") >= 60).then(pl.lit("C"))
      .otherwise(pl.lit("F"))
      .alias("grade")
)

Rename & drop

Pandas
df.rename(columns={"old": "new"})
df.drop(columns=["tmp", "debug"])
df.columns = [c.lower() for c in df.columns]
Polars
df.rename({"old": "new"})
df.drop(["tmp", "debug"])
df.rename({c: c.lower() for c in df.columns})

Sorting & Ranking

Multi-key sorting, null placement control, and stable sort options. Both support per-column sort direction.

Sort

Pandas
df.sort_values("score", ascending=False)

df.sort_values(
    ["dept", "salary"],
    ascending=[True, False],
    na_position="last",
    kind="stable",
)

df.sort_values("score", key=lambda s: s.abs())
Polars
df.sort("score", descending=True)

df.sort(
    ["dept", "salary"],
    descending=[False, True],
    nulls_last=True,
    maintain_order=True,
)

df.sort(pl.col("score").abs(), descending=True)

Rank globally

Pandas
df["rank_dense"] = df["score"].rank(
    method="dense", ascending=False
)
df["rank_pct"] = df["score"].rank(pct=True)
df["row_num"]  = df["score"].rank(
    method="first", ascending=False
).astype(int)
Polars
df.with_columns(
    pl.col("score")
      .rank(method="dense", descending=True)
      .alias("rank_dense"),
    pl.col("score")
      .rank(method="ordinal", descending=True)
      .alias("row_num"),
)

Top-N rows per group

Pandas
top3 = (
    df.sort_values("revenue", ascending=False)
      .groupby("dept")
      .head(3)

top3 = (
    df.sort("revenue", descending=True)

07

GroupBy & Aggregation

Both libraries support multi-column grouping and multi-function aggregation. Polars evaluates all aggregations in parallel across CPU cores. Pandas 3.0 uses named-tuple syntax for clean output column naming.

Basic aggregation

Pandas
df.groupby("dept").agg(
    total_sales = ("sales",  "sum"),
    avg_salary  = ("salary", "mean"),
    headcount   = ("id",     "count"),
    max_score   = ("score",  "max"),
    uniq_cities = ("city",   "nunique"),
).reset_index()
Polars
df.group_by("dept").agg(
    pl.col("sales").sum().alias("total_sales"),
    pl.col("salary").mean().alias("avg_salary"),
    pl.len().alias("headcount"),
    pl.col("score").max().alias("max_score"),
    pl.col("city").n_unique().alias("uniq_cities"),
)

Multiple grouping keys + percentile

Pandas
df.groupby(["region", "product"]).agg(
    revenue = ("amount",   "sum"),
    orders  = ("order_id", "nunique"),
    p90     = ("amount",   lambda x: x.quantile(.9)),
).reset_index()
Polars
df.group_by(["region", "product"]).agg(
    pl.col("amount").sum().alias("revenue"),
    pl.col("order_id").n_unique().alias("orders"),
    pl.col("amount").quantile(.9).alias("p90"),
)

Collect values into a list

Pandas
df.groupby("user_id").agg(
    events = ("event", list),
    tags   = ("tag", lambda x: sorted(x.unique())),
).reset_index()
Polars
df.group_by("user_id").agg(
    pl.col("event").alias("events"),
    pl.col("tag").unique().sort().alias("tags"),
)

Rolling & expanding (row-based)

Pandas
df["ma7"]  = df["value"].rolling(7).mean()
df["ma7c"] = df["value"].rolling(7, min_periods=1).mean()
df["std7"] = df["value"].rolling(7).std()
df["cum"]  = df["value"].expanding().sum()
Polars
df.with_columns(
    pl.col("value").rolling_mean(7).alias("ma7"),
    pl.col("value").rolling_mean(7, min_periods=1).alias("ma7c"),
    pl.col("value").rolling_std(7).alias("std7"),
    pl.col("value").cum_sum().alias("cum"),
)
08

Window Functions

Window functions compute per-row results scoped to a group — without collapsing rows the way groupby does. Polars uses .over("key") as its SQL PARTITION BY. Pandas uses transform, cumsum, shift, and rank on grouped objects.

Rank within group

Pandas
df["rank"] = (
    df.groupby("dept")["salary"]
      .rank(method="dense", ascending=False)
)
df["rank_min"] = (
    df.groupby("dept")["salary"]
      .rank(method="min", ascending=False)
)
df["pct_rank"] = (
    df.groupby("dept")["salary"]
      .rank(pct=True)
)
Polars
df.with_columns(
    pl.col("salary")
      .rank(method="dense", descending=True)
      .over("dept")
      .alias("rank"),
    pl.col("salary")
      .rank(method="min", descending=True)
      .over("dept")
      .alias("rank_min"),
)

Row number within group ROW_NUMBER in SQL

Pandas
df = df.sort_values(
    ["dept", "salary"],
    ascending=[True, False]
)
df["row_num"] = (
    df.groupby("dept").cumcount() + 1
)
Polars
df = df.sort(
    ["dept", "salary"],
    descending=[False, True]
)
df.with_columns(
    pl.col("salary")
      .rank(method="ordinal", descending=True)
      .over("dept")
      .alias("row_num")
)

Lag & Lead shift forward / backward

Pandas
df["prev"] = (
    df.groupby("user_id")["amount"].shift(1)
)
df["next"] = (
    df.groupby("user_id")["amount"].shift(-1)
)
df["lag2"] = (
    df.groupby("user_id")["amount"].shift(2)
)
Polars
df.with_columns(
    pl.col("amount").shift(1)
      .over("user_id").alias("prev"),
    pl.col("amount").shift(-1)
      .over("user_id").alias("next"),
    pl.col("amount").shift(2)
      .over("user_id").alias("lag2"),
)

Period-over-period change delta vs previous row

Pandas
df["delta"] = (
    df.groupby("ticker")["close"].diff(1)
)
df["pct_chg"] = (
    df.groupby("ticker")["close"].pct_change()
)
Polars
df.with_columns(
    pl.col("close").diff(1)
      .over("ticker").alias("delta"),
    (pl.col("close").diff() /
     pl.col("close").shift(1))
      .over("ticker").alias("pct_chg"),
)

Cumulative sum / max / min within group

Pandas
df["running_total"] = (
    df.groupby("user_id")["amount"].cumsum()
)
df["running_max"] = (
    df.groupby("user_id")["amount"].cummax()
)
df["running_min"] = (
    df.groupby("user_id")["amount"].cummin()
)
Polars
df.with_columns(
    pl.col("amount").cum_sum()
      .over("user_id").alias("running_total"),
    pl.col("amount").cum_max()
      .over("user_id").alias("running_max"),
    pl.col("amount").cum_min()
      .over("user_id").alias("running_min"),
)

Group statistics broadcast to every row GROUP BY … JOIN BACK

Pandas
df["dept_avg"] = (
    df.groupby("dept")["salary"].transform("mean")
)
df["dept_total"] = (
    df.groupby("dept")["revenue"].transform("sum")
)
df["dept_count"] = (
    df.groupby("dept")["id"].transform("count")
)
df["dept_max"] = (
    df.groupby("dept")["salary"].transform("max")
)
Polars
df.with_columns(
    pl.col("salary").mean()
      .over("dept").alias("dept_avg"),
    pl.col("revenue").sum()
      .over("dept").alias("dept_total"),
    pl.len()
      .over("dept").alias("dept_count"),
    pl.col("salary").max()
      .over("dept").alias("dept_max"),
)

Rolling window within group

Pandas
df["roll3"] = (
    df.groupby("dept")["revenue"]
      .transform(lambda x: x.rolling(3, min_periods=1).mean())
)
df["roll_sum"] = (
    df.groupby("dept")["revenue"]
      .transform(lambda x: x.rolling(3).sum())
)
Polars
df.with_columns(
    pl.col("revenue")
      .rolling_mean(3, min_periods=1)
      .over("dept").alias("roll3"),
    pl.col("revenue")
      .rolling_sum(3)
      .over("dept").alias("roll_sum"),
)

First & last value in group

Pandas
df["first_buy"] = (
    df.groupby("user_id")["amount"].transform("first")
)
df["last_buy"] = (
    df.groupby("user_id")["amount"].transform("last")
)
Polars
df.with_columns(
    pl.col("amount").first()
      .over("user_id").alias("first_buy"),
    pl.col("amount").last()
      .over("user_id").alias("last_buy"),
)

Quantile & n-tile within group

Pandas
df["p90"] = (
    df.groupby("dept")["score"]
      .transform(lambda x: x.quantile(.9))
)
df["quartile"] = (
    df.groupby("dept")["score"]
      .transform(lambda x: pd.qcut(x, 4, labels=False) + 1)
)
Polars
df.with_columns(
    pl.col("score").quantile(.9)
      .over("dept").alias("p90"),
)
df.with_columns(
    pl.col("score")
      .qcut(4, labels=["Q1","Q2","Q3","Q4"])
      .over("dept").alias("quartile")
)

Multi-partition window Polars

Polars — partition by multiple keys
df.with_columns(
    pl.col("revenue").mean()
      .over(["region", "product_type"])
      .alias("segment_avg"),
    pl.col("revenue").rank(descending=True)
      .over(["region", "product_type"])
      .alias("rank_in_segment"),
)

Polars .over() = SQL PARTITION BY. Every SQL window expression — AVG(salary) OVER (PARTITION BY dept), RANK() OVER (PARTITION BY dept ORDER BY salary DESC) — maps directly to Polars expression syntax. Chain any aggregation before .over() and it becomes a window function.

09

Pivot & Reshape

Reshaping between wide and long format is one of the most common data-wrangling tasks. Pandas has rich pivot support with margins and normalization. Polars is more streamlined. Polars 1.36 added LazyFrame.pivot.

pivot() — simple rotation, no aggregation one value per cell

Pandas — df.pivot()
wide = df.pivot(
    index="date",
    columns="product",
    values="revenue",
)
wide.columns.name = None
wide = wide.reset_index()
Polars — df.pivot()
wide = df.pivot(
    index="date",
    on="product",
    values="revenue",
    aggregate_function=None,
)

pivot_table() — with aggregation handles duplicate cell values

Pandas — pd.pivot_table()
pt = pd.pivot_table(
    df,
    index="region",
    columns="product",
    values="revenue",
    aggfunc="sum",
    fill_value=0,
    margins=True,
    margins_name="Total",
    dropna=True,
).reset_index()
Polars — df.pivot()
pt = df.pivot(
    index="region",
    on="product",
    values="revenue",
    aggregate_function="sum",
    maintain_order=True,
    sort_columns=True,
).fill_null(0)

pivot_table() — multiple values & aggfuncs

Pandas
pt = pd.pivot_table(
    df,
    index="dept",
    columns="year",
    values=["revenue", "cost"],
    aggfunc={
        "revenue": "sum",
        "cost":    "mean",
    },
    fill_value=0,
)
pt.columns = ["_".join(map(str, c)) for c in pt.columns]
Polars — group_by + pivot
agg = df.group_by(["dept", "year"]).agg(
    pl.col("revenue").sum().alias("revenue_sum"),
    pl.col("cost").mean().alias("cost_mean"),
)
wide = agg.pivot(
    index="dept",
    on="year",
    values="revenue_sum",
    aggregate_function="first",
)

melt() / unpivot() — wide → long

The inverse of pivot. Converts multiple value columns into rows — essential before most tidy-data operations.

Pandas — pd.melt()
long = pd.melt(
    df,
    id_vars=["id", "name"],
    value_vars=["q1", "q2", "q3", "q4"],
    var_name="quarter",
    value_name="revenue",
)

long = df.melt(
    id_vars=["id"],
    var_name="metric",
    value_name="value",
)
Polars — df.unpivot()
long = df.unpivot(
    index=["id", "name"],
    on=["q1", "q2", "q3", "q4"],
    variable_name="quarter",
    value_name="revenue",
)

long = df.unpivot(
    index=["id"],
    variable_name="metric",
    value_name="value",
)

stack() / unstack() — hierarchical indexes Pandas only

Pandas
stacked   = df.set_index(["region", "product"]).stack()
unstacked = df.set_index("region").unstack("product")
unstacked.columns = ["_".join(c) for c in unstacked.columns]
stacked   = df.set_index(["region", "product"]).stack(future_stack=True)

crosstab() — frequency / contingency tables Pandas only

Pandas — pd.crosstab()
ct = pd.crosstab(df["dept"], df["grade"])

ct_pct = pd.crosstab(
    df["dept"], df["grade"], normalize="index"
)

ct_val = pd.crosstab(
    df["dept"], df["grade"],
    values=df["salary"],
    aggfunc="mean",
    margins=True,
    margins_name="Overall",
)

Polars 1.36 added LazyFrame.pivot. When the unique column values are known in advance, you can pivot inside a lazy pipeline without materialising first — pass the known column names as values=[...] to the eager pivot call, or use the lazy variant for streaming-compatible reshaping in production pipelines.

Joins & Merges

Polars uses a multi-threaded hash join engine that is typically 3–9× faster than Pandas for large datasets. Both support all major join types; Polars adds native semi, anti, and streaming ASOF joins.

Join typePandasPolars
Innerhow="inner"how="inner"
Left outerhow="left"how="left"
Right outerhow="right"how="right"
Full outerhow="outer"how="full"
Cross joinhow="cross"how="cross"
Semi joindf[df.id.isin(other.id)]how="semi"
Anti joindf[~df.id.isin(other.id)]how="anti"
ASOF joinpd.merge_asof().join_asof()

Standard join

Pandas
result = orders.merge(
    customers,
    on="customer_id",
    how="left",
    suffixes=("_order", "_cust"),
)
result = pd.merge(
    orders, customers,
    left_on="cust_id", right_on="id",
    how="inner",
)
Polars
result = orders.join(
    customers,
    on="customer_id",
    how="left",
    suffix="_cust",
)
result = orders.join(
    customers,
    left_on="cust_id", right_on="id",
    how="inner",
)

ASOF join — match nearest timestamp

Essential for financial tick data, sensor readings, and event-log attribution where events don't align on exact timestamps.

Pandas
result = pd.merge_asof(
    trades.sort_values("ts"),
    quotes.sort_values("ts"),
    on="ts",
    by="ticker",
    direction="backward",
    tolerance=pd.Timedelta("1s"),
)
Polars — streaming ASOF (1.39+)
result = trades.join_asof(
    quotes,
    on="ts",
    by="ticker",
    strategy="backward",
    tolerance="1s",
)

Concatenate DataFrames

Pandas
pd.concat([df1, df2, df3], ignore_index=True)
pd.concat([df1, df2], axis=1)
pd.concat([df1, df2], join="inner", ignore_index=True)

Apply & Custom Functions

Native vectorised expressions should always be the first choice. Fall back to Python UDFs only when no built-in expression covers the case — they sacrifice parallelism and type inference.

Element-wise map

Pandas — .map()
df["score_x2"] = df["score"].map(lambda x: x * 2)
df["label"] = df["code"].map({
    "A": "Active",
    "I": "Inactive",
    "P": "Pending",
})
Polars — map_elements() / replace()
df.with_columns(
    pl.col("score")
      .map_elements(lambda x: x * 2, return_dtype=pl.Float64)
      .alias("score_x2"),
)
mapping = {"A": "Active", "I": "Inactive"}
df.with_columns(pl.col("code").replace(mapping))

Row-wise apply

Pandas — apply(axis=1)
df["full_name"] = df.apply(
    lambda row: f"{row['first']} {row['last']}", axis=1
)
df["result"] = df.apply(
    lambda row: row["a"] * row["b"] + row["c"], axis=1
)
Polars — prefer expressions
df.with_columns(
    (pl.col("first") + pl.lit(" ") + pl.col("last"))
      .alias("full_name"),
    (pl.col("a") * pl.col("b") + pl.col("c"))
      .alias("result"),
)

Series-level batch UDF — map_batches

Operates on the entire column as a Series at once. Faster than map_elements for NumPy-compatible operations since it avoids per-element Python overhead.

Pandas — vectorised NumPy
import numpy as np
df["log_score"] = np.log1p(df["score"])
df["clipped"]  = np.clip(df["val"], 0, 100)
Polars — map_batches()
import numpy as np
df.with_columns(
    pl.col("score")
      .map_batches(lambda s: pl.Series(np.log1p(s.to_numpy())))
      .alias("log_score"),
    pl.col("val").clip(0, 100).alias("clipped"),
)

Prefer native expressions. map_elements serialises to Python for every single element — it disables multi-threading, bypasses type inference, and is typically 10–100× slower than an equivalent vectorised expression. Always exhaust the built-in expression API first.

Polars Lazy API & Streaming

Polars' defining feature. Build a query plan first, let the optimiser push predicates and column selection into the file reader, eliminate dead branches — then execute. With engine="streaming" it processes larger-than-RAM datasets in batches.

scan → transform → collect

Polars Lazy API

result = (
    lf
    .filter(pl.col("year") == 2026)
    .filter(pl.col("amount") > 0)
    .select(["order_id", "customer", "amount"])
    .group_by("customer")
    .agg(pl.col("amount").sum().alias("total"))
    .sort("total", descending=True)
    .head(100)
)

print(result.explain())
df = result.collect()

When to Use Which

The 2026 answer: use both. Polars for heavy lifting; Pandas for the ML/stats ecosystem handoff. They compose cleanly via a shared Apache Arrow memory layer.

Use Pandas 3.0 when…

  • Integrating with scikit-learn, statsmodels, or PyTorch
  • Working in Jupyter notebooks with matplotlib / seaborn
  • Dataset fits comfortably in RAM (< 2 GB)
  • Team already knows Pandas and migration cost is real
  • You need DatetimeTZDtype, CategoricalDtype, or IntervalIndex
  • SQL → Pandas via pd.read_sql() in existing pipelines
  • Statistical methods: .corr(), .autocorr(), .pct_change()
  • pivot_table with margins, normalize, and multi-aggfunc
  • Hierarchical indexes — stack / unstack / MultiIndex

Use Polars 1.x when…

  • Dataset exceeds RAM — use the streaming engine
  • ETL pipelines and data engineering at scale
  • You need parallel CPU execution (joins, groupby, window)
  • Working with Parquet / Arrow data from cloud storage
  • Nested JSON / Struct / List / Array columns
  • Performance is a hard requirement (3–30× faster)
  • Deterministic, reproducible transforms (immutable DataFrames)
  • SQL-style window functions with .over()
  • Lazy query optimisation and predicate pushdown
